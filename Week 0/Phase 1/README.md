Phase 1 – SQL to PySpark
What I Learned:
In this phase, I started learning how to work with PySpark and how it is similar to SQL. I understood how to perform basic operations on data using DataFrames. I practiced:

Selecting columns using select()
Filtering data using filter()
Grouping data using groupBy()
Displaying results using show() I also learned how SQL queries can be converted into PySpark code, which helped me understand both concepts more clearly.
SQL to PySpark Understanding:
During this phase, I connected SQL concepts with PySpark:

SELECT → select()
WHERE → filter()
GROUP BY → groupBy() This made it easier to think in PySpark instead of only SQL.
What I Implemented:
I worked on simple queries like:

Displaying all customer data
Filtering customers based on city
Finding customers with age greater than 25
Selecting only required columns
Counting number of customers in each city
Challenges Faced:
Initially, it was confusing to switch from SQL syntax to PySpark functions
Understanding how groupBy() works took some time
Writing conditions in PySpark felt different compared to SQL
Final Outcome:
By the end of this phase, I became comfortable with basic PySpark operations and understood how SQL logic is applied in PySpark. This gave me a good foundation to move on to more advanced topics like joins and aggregations.
